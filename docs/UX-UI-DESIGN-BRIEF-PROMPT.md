# Промт для создания ТЗ на UX/UI дизайн

Скопируйте шаблон ниже и подставьте данные своего проекта в места, отмеченные квадратными скобками.

---

**Роль:**  
Ты — опытный Senior Product Designer и системный аналитик. Твоя задача — составить подробное техническое задание (ТЗ) на разработку UX/UI дизайна для веб-приложения на основе вводных данных.

## Контекст проекта

- **Название проекта:** [Название]
- **Тип продукта:** [Например: CRM-система, маркетплейс, панель управления серверами, сервис для работы с AR]
- **Целевая аудитория:** [Кто будет пользоваться? Например: технические специалисты, предприниматели, обычные пользователи]
- **Основная бизнес-цель:** [Какую проблему решает приложение?]

## Технические параметры

- **Платформа:** Web ([Desktop-first / Mobile-first / Adaptive])
- **Стек:** [Например: React, TypeScript, Python/Java на бэкенде]
- **Основные функции MVP:**
  - [Функция 1]
  - [Функция 2]
  - [Функция 3]
  - [Функция 4]
  - [Функция 5]

## Требования к структуре ТЗ

Опиши подробно следующие разделы:

1. **Информационная архитектура**
   - Карта страниц
   - Логическая вложенность
   - Связи между разделами

2. **User Flow**
   - Описание пути пользователя для ключевых сценариев
   - Шаги от входа в систему до выполнения целевого действия
   - Альтернативные и ошибочные сценарии

3. **UX-требования**
   - Сетка
   - Логика навигации
   - Поведение интерфейса
   - Пустые состояния
   - Ошибки
   - Загрузка
   - Успешные действия

4. **UI-требования**
   - Общая стилистика: [минимализм / футуризм / строгий бизнес / другое]
   - Предпочтения по цветам
   - Требования к типографике
   - Требования к визуальной иерархии
   - Требования к адаптивности

5. **Состав дизайн-системы (UI Kit)**
   - Кнопки
   - Инпуты
   - Таблицы
   - Селекты
   - Чекбоксы
   - Радиокнопки
   - Табы
   - Модальные окна
   - Тосты
   - Дропдауны
   - Карточки
   - Бейджи
   - Пагинация
   - Сайдбар
   - Хедер
   - Пустые и error-state компоненты

6. **Список конкретных экранов**
   - Перечисли все страницы и состояния, которые нужно отрисовать для передачи в разработку
   - Для каждого экрана укажи ключевые блоки и сценарии

7. **Формат сдачи работы**
   - Макеты в Figma по слоям
   - Кликабельный прототип
   - UI Kit
   - Комментарии для разработчиков
   - Спецификация состояний и интеракций

## Требования к стилю ответа

- Пиши профессионально и структурированно
- Используй русский язык
- Используй таблицы и списки для наглядности
- Не ограничивайся общими фразами: формулируй результат как рабочее ТЗ, пригодное для передачи дизайнеру и команде разработки
- Если каких-то данных не хватает, сначала перечисли недостающие вводные, а затем предложи предварительную версию ТЗ на основе разумных допущений

## Формат итогового результата

Оформи ответ в следующей структуре:

1. Краткое описание продукта
2. Информационная архитектура
3. User Flow
4. UX-требования
5. UI-требования
6. Состав UI Kit
7. Список экранов
8. Формат сдачи
9. Вопросы и допущения
